## 1. ข้อกำหนดของระบบ (System Requirements)

| รายการ | ความต้องการ |
|---|---|
| **ระบบปฏิบัติการ** | Windows 10 / 11 (64-bit) |
| **หน่วยความจำ (RAM)** | ขั้นต่ำ 8 GB (แนะนำ 16 GB ขึ้นไปสำหรับการทำ AI Scoring) |
| **พื้นที่จัดเก็บข้อมูล** | ว่างอย่างน้อย 5 GB (สำหรับตัวโปรแกรมและโมเดล AI) |
| **อินเทอร์เน็ต** | จำเป็นต้องมีเพื่อเชื่อมต่อดึงข้อมูลข่าว (Scraping) |
| **Ollama AI** | ติดตั้งอัตโนมัติหากเลือกติ๊กถูกในขั้นตอนติดตั้งโปรแกรม |

## 2. การติดตั้ง (Installation)

1. ดาวน์โหลดไฟล์และดับเบิลคลิก **`AIEAT_Setup.exe`**
2. ⚠️ **หมายเหตุสำคัญ (Windows SmartScreen):** 
   เนื่องจากโปรแกรมนี้เป็นซอฟต์แวร์ Open Source และไม่ได้เซ็นรับรอง (Code Signing) Windows อาจแสดงหน้าต่างสีฟ้าแจ้งเตือนว่า **"Windows protected your PC"** ในครั้งแรกที่เปิดใช้งาน
   - ให้คลิกที่คำว่า **"More info"**
   - จากนั้นคลิกปุ่ม **"Run anyway"** เพื่อดำเนินการติดตั้งต่อตามปกติ

3. ทำตามขั้นตอนบนหน้าจอติดตั้ง (แนะนำให้ใช้ตำแหน่งการติดตั้งตามค่าเริ่มต้น)
4. เมื่อถึงหน้า **"AI Engine"** คุณจะเห็นตัวเลือก:
   > [✔️] **Install Ollama AI Engine + Typhoon 2.5 Model**
   
   **ให้ติ๊กเลือกหัวข้อนี้** ระบบจะทำการดาวน์โหลดโมเดลภาษา AI แบบ Local (~3-4 GB) ซึ่งจำเป็นอย่างยิ่งสำหรับการประเมินความสำคัญของข่าวและแปลภาษา หากไม่ติดตั้ง ฟีเจอร์ AI จะไม่ทำงาน

5. รอให้ระบบติดตั้งโปรแกรมและโมเดล AI จนเสร็จ (กระบวนการติดตั้ง Typhoon 2.5 อาจใช้เวลา 5-10 นาที ขึ้นอยู่กับความเร็วอินเทอร์เน็ตของคุณ)
6. คลิก **Finish** ถือเป็นการเสร็จสิ้น AIEAT พร้อมใช้งานแล้ว!

> **หมายเหตุ:** หากคุณกดข้ามการติดตั้ง Ollama ในตอนแรก คุณสามารถติดตั้งภายหลังได้โดยการดับเบิลคลิกไฟล์ `Install_AI_Engine.bat` ในโฟลเดอร์ที่ติดตั้งโปรแกรม AIEAT

## 3. วิธีใช้งาน (How to Use)

AIEAT แบ่งการทำงานออกเป็น **โปรไฟล์ (Profiles)** เพื่อแยกหมวดหมู่ข่าวสาร เช่น ข่าวการเงิน ข่าวไอที แต่ละโปรไฟล์จะมีแหล่งข่าวและคีย์เวิร์ดแยกกันอย่างเด็ดขาด

### 📍 ขั้นตอนที่ 1: การเลือก/สร้างโปรไฟล์
1. เปิดโปรแกรม AIEAT จาก Shortcut บน Desktop
2. ที่มุมขวาบนของหน้าจอ ให้คลิกปุ่ม **Profile**
3. เลือกโปรไฟล์ที่ต้องการ (เช่น "Technology & AI") หรือกดสร้างโปรไฟล์ใหม่ หน้ากระดานข่าวจะโหลดข้อมูลเฉพาะของโปรไฟล์นั้นๆ ขึ้นมา

### 📍 ขั้นตอนที่ 2: การเพิ่มแหล่งข่าว (News Sources)
⚠️ **ต้องทำก่อนเริ่มดึงข้อมูลข่าว!** เครื่องมือ Scraper จะดึงข้อมูลเฉพาะจากเว็บไซต์ที่คุณกำหนดไว้เท่านั้น
1. ไปที่เมนู **User Config** ด้านซ้าย
2. เลื่อนลงมาที่หัวข้อ **"News Sources"**
3. คัดลอก URL ของเว็บไซต์ข่าวที่ต้องการ (เช่น `https://www.reuters.com/business/tech`) วางในช่องและกด **Add Source**
4. หากมีเว็บไซต์หลายเว็บ สามารถกด **Import File** นำเข้าไฟล์ `.txt` ที่มีรายการ URL หลายๆ บรรทัดได้เลย

### 📍 ขั้นตอนที่ 3: การตั้งค่าคำค้นหา (Keywords)
⚠️ **สำคัญมาก:** คีย์เวิร์ดคือตัวกรอง หากคุณไม่ได้ตั้งคีย์เวิร์ดเลย **ระบบจะไม่บันทึกข่าวใดๆ เลย**
1. ในหน้า **User Config** เลื่อนไปที่หัวข้อ **"Keywords"**
2. เพิ่มคำค้นหาหลักที่ต้องการติดตาม (เช่น `Nvidia`, `AI`, `Stock`, `อัตราดอกเบี้ย`)
3. ระบบใช้เงื่อนไขแบบ OR - ข่าวจะถูกดึงมาเก็บไว้หากมีคีย์เวิร์ด **คำใดคำหนึ่ง** ที่คุณตั้งไว้ปรากฏในเนื้อหาข่าว

### 📍 ขั้นตอนที่ 4: การดึงข้อมูลข่าว (Scraping)
1. กลับไปที่หน้า **Dashboard** (เมนูซ้ายมือ)
2. คลิกปุ่มสีน้ำเงิน **Start Scraping**
3. ระบบจะเข้าไปยังเว็บไซต์กวาดอ่านเนื้อหาทั้งหมด และคัดเฉพาะข่าวที่ตรงกับ "Keywords" ของคุณลงฐานข้อมูล
4. แถบความคืบหน้าด้านล่างจะคอยอัปเดตสถานะการทำงาน

### 📍 ขั้นตอนที่ 5: การให้คะแนนด้วย AI (Batch Scoring)
เมื่อดึงข่าวมาแล้ว บางครั้งอาจมีข่าวจำนวนมากเกินกว่าจะอ่านไหว
1. ในหน้า Dashboard คลิกปุ่ม **Batch Score** ด้านบน
2. ระบบ AI จะทำการอ่านสรุปเนื้อหาและให้ **คะแนนความเกี่ยวข้อง (Relevance Score) ตั้งแต่ 1-10** (ใช้เวลาประมาณ 5-10 วินาที/บทความ)
3. เมื่อเสร็จสิ้น สามารถกดเมนู **Filter** เพื่อกรองดูเฉพาะข่าวที่มีคะแนน "High Relevance (5+)" ได้

### 📍 ขั้นตอนที่ 6: การเปลี่ยนสไตล์และการแปลภาษา
1. หากเจอข่าวที่สนใจ กด **View Detail**
2. คุณจะเห็นเนื้อหาข่าวต้นฉบับ หากต้องการแปล ให้กดปุ่ม **Translate**
3. AI จะแปลและเรียบเรียงเป็นภาษาไทยให้ตามรูปแบบที่กำหนดไว้ใน **Style Settings** (เมนูซ้ายมือ)
4. *เคล็ดลับ:* ใน Style Settings คุณสามารถเปลี่ยนรูปแบบการแปลได้ เช่น
   - **Facebook Post:** แปลงข่าวเป็นโพสต์พร้อมแฮชแท็ก โทนสนุกสนาน
   - **Executive Summary:** สรุปสาระสำคัญสั้นๆ เป็นทางการ สำหรับผู้บริหาร

## 4. กรณีการใช้งานจริง (Use Cases)

AIEAT สามารถปรับแต่งให้เข้ากับรูปแบบการทำงานได้หลากหลาย:

### 💼 กรณีที่ 1: นักวิเคราะห์การเงิน (Finance Analyst)
- **โจทย์:** ต้องเกาะติดข่าวเศรษฐกิจโลกทุกเช้าเพื่อส่งบรีฟให้ผู้จัดการกองทุน
- **การตั้งค่า:** สร้างโปรไฟล์ "Finance Markets" แอดเว็บ Bloomberg, Reuters 
- **คีย์เวิร์ด:** `Interest Rate`, `Fed`, `Inflation`, `S&P 500`
- **วิธีใช้:** กด Scrape ดึงข่าวเมื่อคืนทั้งหมด -> กด Batch Score เพื่อกรองเอาแต่ข่าวคะแนน 8-10 -> เข้ามาอ่านฉบับแปลสรุป (ตั้ง Style เป็น Executive Summary) ประหยัดเวลาอ่านได้กว่า 2 ชั่วโมงต่อวัน

### 📱 กรณีที่ 2: นักทำคอนเทนต์โซเชียลมีเดีย (Social Media Manager)
- **โจทย์:** หักโหมหาคอนเทนต์ไอทีใหม่ๆ มาลงเพจ Facebook ทุกวัน
- **การตั้งค่า:** สร้างโปรไฟล์ "Tech Trends" แอดเว็บ The Verge, TechCrunch
- **คีย์เวิร์ด:** `Generative AI`, `Apple`, `Gadget`
- **วิธีใช้:** หลังจากดึงข่าวและเจอเรื่องน่าสนใจ ตั้งค่า AI Style ให้เป็น **Facebook Post (Conversational)** -> กด Translate AIEAT จะดึงใจความสำคัญมาเขียนเป็นแคปชั่น Facebook ภาษาไทยพร้อมแฮชแท็ก สามารถก๊อปปี้ไปโพสต์ได้แทบจะทันที

### 🏛️ กรณีที่ 3: ทีมประชาสัมพันธ์รักษาภาพลักษณ์องค์กร (PR Team)
- **โจทย์:** คอยเฝ้าระวังว่ามีสำนักข่าวไหนพูดถึงบริษัทของตัวเองบ้างหรือไม่
- **การตั้งค่า:** สร้างโปรไฟล์ "Brand Monitor" แอดเว็บไซต์ข่าวทั้งในและต่างประเทศ
- **คีย์เวิร์ด:** ใส่ชื่อบริษัทของตัวเอง คู่แข่ง และชื่อผู้บริหาร
- **วิธีใช้:** ใช้ระบบ Scraper ทุกเช้า หากพบว่ามีการเคลื่อนไหวชื่อบริษัท ก็จะถูกคัดกรองเข้ามาใน Dashboard ให้รับรู้และตั้งรับได้ทันท่วงที

## 5. การแก้ปัญหาเบื้องต้น (Troubleshooting)

**ปัญหา: แจ้งเตือน "AI Server Offline" หรือ "Ollama not running"**
- สาเหตุ: AI ไม่ได้ถูกรันบน Windows
- วิธีแก้: ค้นหาคำว่า `Ollama` ในช่องค้นหา (Start Menu) ของ Windows แล้วกดเปิด จะเห็นไอคอนตัวลามาสีขาวขึ้นที่ขวาล่างของจอ รอประมาณ 15 วินาทีแล้วใช้งาน AIEAT อีกครั้ง

**ปัญหา: เครื่องมือจำกัดการดึงข่าวได้ 0 บทความตลอดเวลา**
- สาเหตุ: ดึงจาก URL ผิด หรือตั้งคีย์เวิร์ดแคบไป
- วิธีแก้: ตรวจสอบว่า `Keywords` ในหน้าตั้งค่ามีคำกว้างๆ อยู่บ้าง (เช่น `หุ้น` แทนที่จะใส่ตัวย่อบริษัทแปลกๆ อย่างเดียว) และเช็กว่า URL ของข่าวใช้งานได้จริงผ่าน Browser

## การจัดการข้อมูลและการสำรองข้อมูล (Data & Backup)
**ตำแหน่งเก็บข้อมูล:** บทความและการตั้งค่าทั้งหมดถูกเก็บไว้ในที่เดียวคือไฟล์ `data/aieat_news.db` ในโฟลเดอร์ที่ติดตั้งโปรแกรม  
**การสำรองข้อมูล (Backup):** เพียงแค่คัดลอกไฟล์ `aieat_news.db` เก็บไว้ หากต้องการกู้คืน ก็นำไฟล์เดิมมาวางทับได้เลย

## ความเป็นส่วนตัวและความปลอดภัย (Privacy Policy)
AIEAT เป็นแอปพลิเคชันแบบ **Local-first 100%**
- ข่าวทั้งหมดที่คุณดึงมาจะถูกเก็บไว้ในเครื่องของคุณเท่านั้น
- การประมวลผลให้คะแนนและแปลภาษา ดำเนินการโดยโมเดล Ollama ภายในเครื่อง
- **ไม่มีการส่งข้อมูลใดๆ** ไปยังระบบคลาวด์ภายนอก หรือ OpenAI

## ข้อจำกัดและประสิทธิภาพการทำงาน (Limitations & Performance)
- **เว็บไซต์บล็อกการดึงข้อมูล:** บางเว็บไซต์มีระบบป้องกันบอท (เช่น Cloudflare) หรือต้องล็อกอินเข้าระบบ หากพบว่าเว็บไหนดึงข่าวไม่ได้เลยตลอดเวลา แนะนำให้เปลี่ยนไปใช้ลิงก์ RSS ของเว็บนั้นแทน
- **ความเร็วและขนาดโมเดล AI:** โมเดล Typhoon 2.5 ค่อนข้างกินทรัพยากรเครื่อง หากเครื่องไม่มีการ์ดจอแยก (GPU) การแปลภาษาเต็มบทความอาจใช้เวลา 1-2 นาที
- **จำนวนแหล่งข่าว:** แนะนำให้ใส่แหล่งข่าวคุณภาพเพียง 10-15 เว็บต่อ 1 โปรไฟล์ เพื่อความรวดเร็วในการทำงานของ Scraper

## การอัปเดตและการถอนการติดตั้ง (Update & Uninstall)
- **การอัปเดต:** ดาวน์โหลด `AIEAT_Setup.exe` เวอร์ชันใหม่จากหน้า GitHub Releases และติดตั้งทับที่อยู่เดิมได้เลย ข้อมูลเดิมจะไม่หายไป
- **การถอนการติดตั้ง:** ไปที่ Windows Settings -> Apps -> AIEAT News Dashboard แล้วกด Uninstall *(หมายเหตุ: ฐานข้อมูลจะถูกลบทิ้งไปด้วย กรุณาสำรองไฟล์ db ก่อนถ้าต้องการเก็บข่าวไว้)*

## การเข้าถึงอย่างเท่าเทียม (Accessibility)
หน้าจอของ AIEAT รองรับการใช้คีย์บอร์ดนำทาง (ปุ่ม Tab) สำหรับผู้มีปัญหาด้านการใช้เมาส์ และออกแบบให้มีสีสันที่มีคอนทราสต์ชัดเจน อ่านง่ายสบายตา
